Lo standard di input e output è il medesimo.

Le variabili sulla stessa riga sono separate da spazio, senza lo spazio dopo l'ultima variabile.
A ->
A

A B C ->
A B C

Se ci sono vari array sulla stessa riga devono avere le stesse dimensioni.
Se le dimensioni sono [d1][d2]...[dk], allora vengono create d1*d2*...*dk righe 
e in ognuna di queste, separati da uno spazio come per le variabili, ci sono 
ordinatamente i valori delle relative entrate degli array. 
Quindi se M[2][3] N[2][3] sono due array allora:
M[][] N[][] ->
M[0][0] N[0][0]
M[0][1] N[0][1]
M[1][0] N[1][0]
M[1][1] N[1][1]

Un singolo array viene invece stampato seguendo regole differenti. 
In particolare se ha dimensioni [d1][d2][d3]...[dk] allora vengono create 
d1*d2*...*d(k-1) righe e in ognuna di queste sono stampati le relative dk 
entrate dell'array, separate da uno spazio.
Se però l'array è di tipo char allora gli spazi vengono omessi.
Questi criteri sono sfruttati per gestire bene la scrittura di matrici o di stringhe.

Quindi se M[2][3] è un array non di char allora:
M[][] ->
M[0][0] M[0][1]
M[1][0] M[1][1]

Se invece è un array di char:
M[][] ->
M[0][0]M[0][1]
M[1][0]M[1][1]
